⚔️ 3.2 克隆分身

完整克隆 OpenClaw 记忆，创建你的 AI 分身 | 西门吹雪&掠夺者引擎强力制作

🎯 目标：
在新机器上创建一个完全继承记忆的 OpenClaw 分身，拥有相同的性格、知识和工作方式。

📋 克隆前的准备

1. 确认记忆仓库

首先确认你的记忆已经备份到 GitHub：

cd ~/.openclaw/workspace
git remote -v

应该看到类似：

origin  https://github.com/你的用户名/openclaw-memory.git (fetch)
origin  https://github.com/你的用户名/openclaw-memory.git (push)

2. 记忆文件结构

完整的记忆系统包含以下文件：

workspace/
├── MEMORY.md              # 长期记忆
├── SOUL.md                # 性格和行为准则
├── USER.md                # 关于你的信息
├── AGENTS.md              # 工作规则
├── TOOLS.md               # 工具配置
├── IDENTITY.md            # 身份信息
├── HEARTBEAT.md           # 心跳检测规则
├── RULES.md               # 自定义规则
└── memory/
    ├── YYYY-MM-DD.md      # 每日记录
    ├── knowledge/         # 知识库
    ├── projects/          # 项目记录
    └── conversations/     # 重要对话记录

🚀 完整克隆步骤

1在新机器上安装 OpenClaw

# 安装 Node.js（如果还没有）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 OpenClaw
npm install -g openclaw

# 验证安装
openclaw --version

2初始化 OpenClaw

# 启动 OpenClaw（会自动创建 workspace）
openclaw gateway start

# 等待启动完成
openclaw gateway status

3克隆记忆仓库

# 进入 workspace
cd ~/.openclaw/workspace

# 备份原有文件（如果有）
mkdir -p backup
mv MEMORY.md SOUL.md USER.md backup/ 2>/dev/null || true

# 克隆记忆仓库
git clone https://github.com/你的用户名/openclaw-memory.git temp

# 复制所有文件到 workspace
cp -r temp/* .
cp -r temp/.git .

# 删除临时目录
rm -rf temp

# 验证文件
ls -la

4配置 Git（如果需要推送）

# 配置 Git 用户信息
git config user.name "你的名字"
git config user.email "你的邮箱"

# 测试连接
git remote -v
git pull origin main

5重启 OpenClaw

# 重启以加载新的记忆文件
openclaw gateway restart

# 等待启动
sleep 5

# 检查状态
openclaw gateway status

6验证克隆成功

发送第一条消息测试分身是否继承了记忆：

# 通过 Telegram 或其他方式发送消息
"你还记得我们之前做的螳螂营销云项目吗？"

# 如果分身回答正确，说明记忆继承成功！

🔧 一键克隆脚本

为了简化流程，可以使用这个一键脚本：

#!/bin/bash

# OpenClaw 分身克隆脚本
# 使用方法：bash clone-openclaw.sh https://github.com/你的用户名/openclaw-memory.git

REPO_URL=$1

if [ -z "$REPO_URL" ]; then
    echo "❌ 错误：请提供记忆仓库 URL"
    echo "使用方法：bash clone-openclaw.sh https://github.com/你的用户名/openclaw-memory.git"
    exit 1
fi

echo "🚀 开始克隆 OpenClaw 分身..."

# 1. 检查 OpenClaw 是否已安装
if ! command -v openclaw &> /dev/null; then
    echo "📦 安装 OpenClaw..."
    npm install -g openclaw
fi

# 2. 启动 OpenClaw
echo "🔧 启动 OpenClaw..."
openclaw gateway start
sleep 5

# 3. 进入 workspace
cd ~/.openclaw/workspace

# 4. 备份原有文件
echo "💾 备份原有文件..."
mkdir -p backup
mv MEMORY.md SOUL.md USER.md AGENTS.md backup/ 2>/dev/null || true

# 5. 克隆记忆仓库
echo "📥 克隆记忆仓库..."
git clone "$REPO_URL" temp

# 6. 复制文件
echo "📋 复制记忆文件..."
cp -r temp/* .
cp -r temp/.git .
rm -rf temp

# 7. 重启 OpenClaw
echo "🔄 重启 OpenClaw..."
openclaw gateway restart
sleep 5

# 8. 验证
echo ""
echo "✅ 克隆完成！"
echo ""
echo "📊 记忆文件："
ls -lh MEMORY.md SOUL.md USER.md AGENTS.md 2>/dev/null
echo ""
echo "📁 记忆目录："
ls -lh memory/
echo ""
echo "🎯 下一步："
echo "1. 发送消息测试分身是否继承了记忆"
echo "2. 如果需要推送更新，配置 Git："
echo "   git config user.name '你的名字'"
echo "   git config user.email '你的邮箱'"
💡 使用方法： 
bash clone-openclaw.sh https://github.com/你的用户名/openclaw-memory.git

❌ 常见错误与解决方案

错误 1：克隆后分身没有记忆

现象：分身回答"我不记得"或"我没有这方面的信息"
✅ 解决方案： 
# 1. 检查文件是否存在
cd ~/.openclaw/workspace
ls -lh MEMORY.md SOUL.md USER.md

# 2. 检查文件内容
cat MEMORY.md | head -20

# 3. 检查 AGENTS.md 是否正确
grep "MEMORY.md" AGENTS.md

# 4. 重启 OpenClaw
openclaw gateway restart

# 5. 清除缓存（如果需要）
rm -rf ~/.openclaw/cache/*
openclaw gateway restart

错误 2：Git 克隆失败

现象：fatal: could not read Username 或 Permission denied
✅ 解决方案： 
# 方案 1：使用 HTTPS + Token
git clone https://你的token@github.com/你的用户名/openclaw-memory.git

# 方案 2：使用 SSH
git clone git@github.com:你的用户名/openclaw-memory.git

# 方案 3：手动下载
wget https://github.com/你的用户名/openclaw-memory/archive/refs/heads/main.zip
unzip main.zip
cp -r openclaw-memory-main/* ~/.openclaw/workspace/

错误 3：文件权限问题

现象：Permission denied 或无法读取文件
✅ 解决方案： 
# 修复文件权限
cd ~/.openclaw/workspace
chmod 644 *.md
chmod 755 memory/
chmod 644 memory/*.md
chmod 755 scripts/
chmod 755 scripts/*.sh

错误 4：记忆文件格式错误

现象：OpenClaw 启动失败或报错
✅ 解决方案： 
# 检查文件编码
file MEMORY.md SOUL.md USER.md

# 应该显示：UTF-8 Unicode text
# 如果不是，转换编码：
iconv -f GBK -t UTF-8 MEMORY.md > MEMORY.md.new
mv MEMORY.md.new MEMORY.md

# 检查文件是否有语法错误
cat MEMORY.md | grep -E "^#|^-|^[0-9]"

错误 5：分身性格不对

现象：分身的回答风格和原来不一样
✅ 解决方案： 
# 1. 检查 SOUL.md 是否正确
cat SOUL.md

# 2. 检查 IDENTITY.md
cat IDENTITY.md

# 3. 确认 AGENTS.md 中的启动流程
grep -A 10 "启动流程" AGENTS.md

# 4. 重新克隆（如果文件损坏）
cd ~/.openclaw/workspace
rm -rf *
git clone https://github.com/你的用户名/openclaw-memory.git .
openclaw gateway restart

🔄 保持分身同步

方案 1：手动同步

# 在原机器上推送更新
cd ~/.openclaw/workspace
git add .
git commit -m "更新记忆 $(date +%Y-%m-%d)"
git push

# 在分身机器上拉取更新
cd ~/.openclaw/workspace
git pull origin main
openclaw gateway restart

方案 2：自动同步（推荐）

在分身机器上设置定时拉取：

# 创建同步脚本
cat > ~/.openclaw/workspace/scripts/sync-memory.sh << 'EOF'
#!/bin/bash
cd ~/.openclaw/workspace
git pull origin main
if [ $? -eq 0 ]; then
    echo "✅ 记忆同步成功"
    openclaw gateway restart
else
    echo "❌ 记忆同步失败"
fi
EOF

chmod +x ~/.openclaw/workspace/scripts/sync-memory.sh

# 设置 cron（每小时同步一次）
crontab -e
# 添加：
0 * * * * bash ~/.openclaw/workspace/scripts/sync-memory.sh >> ~/.openclaw/sync.log 2>&1

🎯 高级技巧

1. 多机器同步

如果你有多台机器，可以让它们共享同一个记忆仓库：

# 机器 A（主机）
cd ~/.openclaw/workspace
git push origin main

# 机器 B（分身 1）
cd ~/.openclaw/workspace
git pull origin main

# 机器 C（分身 2）
cd ~/.openclaw/workspace
git pull origin main

2. 分支管理

为不同的分身创建不同的分支：

# 创建分身专用分支
git checkout -b clone-1
git push origin clone-1

# 在分身机器上
git clone -b clone-1 https://github.com/你的用户名/openclaw-memory.git

3. 选择性克隆

如果只想克隆部分记忆：

# 只克隆知识库
git clone --depth 1 --filter=blob:none --sparse https://github.com/你的用户名/openclaw-memory.git
cd openclaw-memory
git sparse-checkout set memory/knowledge/

# 只克隆项目记录
git sparse-checkout set memory/projects/
⚠️ 注意事项：
  • 克隆前确保原机器的记忆已经推送到 GitHub
  • 分身机器需要配置相同的 API Keys 和 Tokens
  • 如果使用 Telegram，需要重新连接 Bot
  • 定期同步记忆，避免分身和原机器记忆不一致
  • 敏感信息（密码、Token）不要提交到 GitHub
