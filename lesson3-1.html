3.1 使用心得 - 自动化录单系统

使用 OpenClaw + Playwright 实现网页自动化录单的完整流程

🎯 项目背景

需要在螳螂营销云系统中批量录入学员信息，手动操作效率低且容易出错。通过 OpenClaw 的 Playwright 技能实现自动化录单。

目标网站：https://yixue.n5.bjmantis.cn/

核心功能：自动登录 → 导航到报名页面 → 填写学员信息 → 提交

📋 完整流程

步骤 1：安装 Playwright 技能

# 使用 ClawHub 安装
clawhub install playwright

步骤 2：创建项目结构

workspace/
├── ludan/              # 录单项目文件夹
│   ├── minzu.json      # 56个民族数据
│   ├── diqu.json       # 34个省级行政区数据
│   └── matcher.js      # 模糊匹配函数
├── photo/              # 截图存放目录
└── scripts/            # 自动化脚本

步骤 3：建立数据库

民族数据（minzu.json）：

  • 包含 56 个民族完整名称
  • 支持模糊匹配：壮 → 壮族，维吾尔 → 维吾尔族

地区数据（diqu.json）：

  • 包含 34 个省级行政区
  • 支持模糊匹配：广西 → 广西壮族自治区，内蒙 → 内蒙古自治区

步骤 4：编写自动化脚本

核心技术点：

  • 模拟人类操作：随机延迟（500-1500ms）+ 慢速输入（delay: 80-120ms）
  • 稳定性优化：使用 slowMo: 100 减慢操作速度
  • 等待策略：waitForTimeout 确保页面加载完成
  • 截图调试：每个关键步骤保存截图到 photo/ 目录
// 随机延迟函数
const humanDelay = (min = 500, max = 1500) => {
  return new Promise(resolve => 
    setTimeout(resolve, Math.random() * (max - min) + min)
  );
};

// 登录流程
await page.click('input[placeholder*="用户名"]');
await humanDelay(300, 600);
await page.type('input[placeholder*="用户名"]', 'username', { delay: 100 });
await page.click('input[type="checkbox"]');  // 勾选协议
await page.click('button:has-text("登")');

🔍 关键问题与解决方案

问题 1：登录失败

原因：忘记勾选"我已阅读并同意"复选框

解决：在点击登录前添加 await page.click('input[type="checkbox"]')

问题 2：找不到按钮

原因：按钮文字中间有空格（"报 名" 而不是 "报名"）

解决：使用 button:has-text("报 名") 精确匹配

问题 3：截图上传慢

原因：每次都推送到 GitHub Pages，需要等待部署

解决：截图保存到本地 photo/ 目录，直接查看

💡 最佳实践

1. 项目组织

  • ludan/ - 业务数据和匹配逻辑
  • photo/ - 调试截图（不提交到 Git）
  • scripts/ - 自动化脚本

2. 模拟人类操作

  • 随机延迟：避免被识别为机器人
  • 慢速输入：每个字符间隔 80-120ms
  • 点击前等待：humanDelay(300, 600)

3. 调试技巧

  • 每个关键步骤截图：方便定位问题
  • 打印页面文本：检查是否加载成功
  • 查找所有按钮：page.$$eval('button', ...)

4. 数据管理

  • 使用 JSON 存储静态数据（民族、地区）
  • 编写匹配函数支持模糊查询
  • 导出为模块方便复用
