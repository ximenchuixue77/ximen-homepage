🎯 系统概述

螳螂营销云API录单系统是一个完全基于API的自动化录单解决方案，绕过了浏览器自动化的复杂性，直接调用后端API完成录单操作。

核心优势

  • ✅ 100%可靠 - 不受页面变化影响
  • ✅ 速度快 - 直接API调用，无需等待页面加载
  • ✅ 易维护 - 代码简洁，逻辑清晰
  • ✅ 可扩展 - 支持批量录单、Excel导入等

2. 核心接口

1初始化基础数据

接口：/syncApi/initOrderBase

作用：获取校区、商品、考期、支付方式等基础数据

返回：

  • 校区列表（campusList）
  • 商品列表（commodityList）- 包含商品ID、版本号、规格ID
  • 考期列表（baseExamList）
  • 支付方式（baseCategoryList）
  • 费用类型（baseFeeTypeList）

2创建订单

接口：/syncApi/syncOrder

作用：提交订单数据，完成录单

必需参数：

  • 客户ID（mspCustomerUuid）
  • 商品ID（commodityId）
  • 商品版本号（commodityVersion）
  • 商品规格ID（commoditySpecesId）- 如果公司开启了规格功能
  • 金额信息（totalAmount、paidAmount、oweAmount）
  • 校区ID（campusId）
  • 考期ID（baseExamId）

🔧 开发步骤

步骤1：测试API连接

const axios = require('axios');
const crypto = require('crypto');

// 配置
const config = {
  baseURL: 'https://mcapi5.bjmantis.cn/hc-api',
  apiKey: '你的密钥',
  companyId: 5752
};

// 生成token
function generateToken(time) {
  return crypto.createHash('md5')
    .update(config.apiKey + time)
    .digest('hex');
}

// 测试连接
async function testConnection() {
  const time = Date.now();
  const token = generateToken(time);
  
  const response = await axios.post(
    `${config.baseURL}/syncApi/initOrderBase`,
    { time, token, companyId: config.companyId }
  );
  
  console.log('连接成功！', response.data);
}

步骤2：获取基础数据

调用初始化接口，获取所有基础数据并保存到本地：

// 保存基础数据
fs.writeFileSync(

步骤3：查找商品和校区

// 查找商品
function findCommodity(name) {
  const baseData = JSON.parse(
    fs.readFileSync('base-data.json')
  );
  
  return baseData.commodityList.find(
    c => c.name.includes(name)
  );
}

// 查找校区
function findCampus(name) {
  const baseData = JSON.parse(
    fs.readFileSync('base-data.json')
  );
  
  return baseData.campusList.find(
    c => c.name.includes(name)
  );
}

步骤4：创建订单

async function createOrder(studentInfo) {
  const time = Date.now();
  const token = generateToken(time);
  
  // 查找商品和校区
  const commodity = findCommodity(studentInfo.commodityName);
  const campus = findCampus(studentInfo.campusName);
  
  // 组装订单数据
  const orderData = {
    time, token,
    companyId: config.companyId,
    syncOrderList: [{
      mspCustomerUuid: studentInfo.customerUuid,
      totalAmount: studentInfo.amount,
      paidAmount: studentInfo.amount,
      oweAmount: 0,
      campusId: campus.id,
      orderDetailList: [{
        commodityId: commodity.id,
        commodityVersion: commodity.currentVersion,
        commoditySpecesId: commodity.specesRespDTOList?.[0]?.id,
        // ... 其他字段
      }],
      ordePayList: [/* 支付信息 */]
    }]
  };
  
  // 提交订单
  const response = await axios.post(
    `${config.baseURL}/syncApi/syncOrder`,
    orderData
  );
  
  return response.data;
}

⚠️ 重要注意事项

1. 商品规格问题

如果公司开启了商品规格功能，所有商品都必须传递规格ID（commoditySpecesId），否则会报错：

"第01个主订单的第1个子订单没有传递正确的规格信息！"

解决方法：从基础数据中获取商品的 specesRespDTOList[0].id

2. 时间格式

所有时间字段必须使用北京时间（UTC+8），格式：YYYY-MM-DD HH:mm:ss

const beijingTime = new Date(
  now.getTime() + (8 * 60 * 60 * 1000)
);
const dateTimeStr = beijingTime
  .toISOString()
  .slice(0, 19)
  .replace('T', ' ');

3. 订单ID范围

thirdOrderId 必须是 int 类型（-2147483648 到 2147483647），不能使用时间戳：

// ❌ 错误：时间戳太大
const thirdOrderId = Date.now(); // 1773396644943

// ✅ 正确：使用随机数
const thirdOrderId = Math.floor(
  Math.random() * 2000000000
);

🔒 安全措施

// 不在代码中硬编码密钥

  apiKey: process.env.MANTIS_API_KEY,

2. 权限控制

  • 超级管理员：查看所有信息
  • 录单员：只能录单
  • 其他人：无权限

3. 敏感信息过滤

在回复前自动检查用户身份，非管理员看不到：

  • API密钥
  • 服务器地址
  • 配置信息
  • 错误详情
