2.2 端口占用

OpenClaw 常用端口及占用解决方案

🚪 OpenClaw 常用端口

端口 用途 说明
18789 Gateway 主端口 默认 HTTP 服务端口
18790 WebSocket 端口 实时通信
5700 QQ Bot 端口 go-cqhttp 默认端口
3000 Web UI 端口 可选的 Web 界面

❌ 问题：端口被占用

错误信息：

Error: listen EADDRINUSE: address already in use :::18789

✅ 解决方案 1：查找并关闭占用进程

# 查看端口占用
lsof -i :18789

# 或使用 netstat
netstat -tulpn | grep 18789

# 杀死占用进程（替换 PID）
kill -9 [PID]

# 重启 Gateway
openclaw gateway restart

✅ 解决方案 2：修改端口

# 编辑 Gateway 配置
nano ~/.openclaw/gateway.json

# 修改端口
{
  "port": 18791,
  "host": "127.0.0.1"
}

# 重启 Gateway
openclaw gateway restart

🔍 检查端口状态

# 查看所有监听端口
netstat -tulpn | grep LISTEN

# 查看 OpenClaw 相关端口
lsof -i -P | grep openclaw

# 测试端口是否可用
nc -zv 127.0.0.1 18789

🛡️ 防火墙配置

如果需要外部访问 OpenClaw，需要开放端口：

# Ubuntu/Debian (ufw)
sudo ufw allow 18789/tcp
sudo ufw reload

# CentOS/RHEL (firewalld)
sudo firewall-cmd --add-port=18789/tcp --permanent
sudo firewall-cmd --reload

⚠️ 安全提醒：不建议将 Gateway 暴露到公网，建议只绑定 127.0.0.1

💡 最佳实践：
  • 使用默认端口 18789，避免冲突
  • 只绑定 127.0.0.1，不暴露到公网
  • 定期检查端口占用情况
  • 使用反向代理（Nginx）处理外部访问
