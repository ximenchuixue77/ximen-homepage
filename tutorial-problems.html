第二课：常见问题解决

遇到问题不要慌，这里有解决方案

❌ 问题 1：安装失败，提示权限错误

错误信息：

Error: EACCES: permission denied

✅ 解决方案 1：使用 sudo（不推荐）

sudo npm install -g openclaw

✅ 解决方案 2：修改 npm 全局目录（推荐）

mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g openclaw

❌ 问题 2：Gateway 启动失败

错误信息：

Error: listen EADDRINUSE: address already in use :::18789

✅ 解决方案：检查并修改端口

# 查看端口占用
lsof -i :18789

# 杀死占用进程
kill -9 [PID]

# 或者修改配置文件
nano ~/.openclaw/gateway.json
# 修改 "port": 18789 为其他端口，如 18790

❌ 问题 3：API Key 无效

错误信息：

Error: Invalid API key

✅ 解决方案：检查并重新配置

# 查看当前配置
cat ~/.openclaw/config.json

# 重新编辑
nano ~/.openclaw/config.json

# 确保格式正确：
{
  "models": {
    "default": "claude-sonnet-4",
    "apiKey": "sk-ant-your-key-here"
  }
}

# 重启 Gateway
openclaw gateway restart

❌ 问题 4：Telegram Bot 无响应

症状：发送消息后 bot 没有回复

✅ 解决方案：检查配置和日志

# 1. 检查 Gateway 是否运行
openclaw gateway status

# 2. 查看日志
openclaw gateway logs -f

# 3. 检查 Telegram 配置
cat ~/.openclaw/config.json | grep -A 5 telegram

# 4. 确认 User ID 是否正确
# 在 Telegram 搜索 @userinfobot 获取你的 User ID

# 5. 重启 Gateway
openclaw gateway restart

❌ 问题 5：技能安装失败

错误信息：

Error: Skill not found

✅ 解决方案：检查技能名称和网络

# 1. 搜索技能确认名称
clawhub search memory

# 2. 检查网络连接
ping clawhub.com

# 3. 清除缓存重试
npm cache clean --force
clawhub install memory-master

# 4. 手动从 GitHub 安装
git clone https://github.com/openclaw/skills.git
cp -r skills/skills/[技能名] ~/.openclaw/workspace/skills/

❌ 问题 6：内存不足

错误信息：

JavaScript heap out of memory

✅ 解决方案：增加 Node.js 内存限制

# 临时增加内存
export NODE_OPTIONS="--max-old-space-size=4096"

# 永久设置（添加到 ~/.bashrc）
echo 'export NODE_OPTIONS="--max-old-space-size=4096"' >> ~/.bashrc
source ~/.bashrc

# 重启 Gateway
openclaw gateway restart

❌ 问题 7：更新后无法启动

症状：更新 OpenClaw 后 Gateway 无法启动

✅ 解决方案：清理并重新初始化

# 1. 备份工作空间
cp -r ~/.openclaw/workspace ~/openclaw-backup

# 2. 停止 Gateway
openclaw gateway stop

# 3. 清理旧版本
npm uninstall -g openclaw
npm cache clean --force

# 4. 重新安装
npm install -g openclaw

# 5. 恢复工作空间
cp -r ~/openclaw-backup/* ~/.openclaw/workspace/

# 6. 启动 Gateway
openclaw gateway start
